ДЕФЕКТ №1

Название: Неэффективная реализация метода contains
Статус: NEW
Приоритет: Major 
Компонент: StringUtils.contains()

Описание:
Метод contains использует string.index() с try-except для проверки наличия символа.
Это неоптимально, так как проще и быстрее использовать оператор `in`.

Текущая реализация:
    try:
        res = string.index(symbol) > -1
    except ValueError:
        pass
    return res

Рекомендуемая реализация:
    return symbol in string

Как воспроизвести:
1. Вызвать contains("some string", "some")
2. Ожидается: True
3. Фактически: True (но через более сложный путь)

Почему это проблема:
- При каждом вызове создается исключение, если символ не найден
- Это дорогостоящая операция
- Метод будет медленным при обработке больших объемов данных

ДЕФЕКТ №2

Название: Избыточный вызов contains в delete_symbol
Статус: NEW
Приоритет: Minor 
Компонент: StringUtils.delete_symbol()

Описание:
В методе delete_symbol сначала вызывается contains для проверки наличия символа,
а затем вызывается replace. Это избыточно, так как replace сам обрабатывает
случай отсутствия символа.

Текущая реализация:
    if self.contains(string, symbol):
        string = string.replace(symbol, "")
    return string

Рекомендуемая реализация:
    return string.replace(symbol, "")

ДЕФЕКТ №3

Название: Отсутствует проверка типов входных данных
Статус: NEW
Приоритет: Major
Компонент: Все методы класса StringUtils

Описание:
При передаче некорректных типов данных (None, int, list) методы 
выбрасывают исключения с неинформативными сообщениями.

Как воспроизвести:
1. Вызвать любой метод с None:
   - capitalize(None)
   - trim(None)
   - contains(None, "a")
   - delete_symbol(None, "a")
2. Ожидается: ValueError с сообщением "Ожидается строка"
3. Фактически: AttributeError или TypeError

Почему это проблема:
- Пользователь не понимает, что пошло не так
- Сообщения об ошибках неинформативны
- Усложняется отладка

ДЕФЕКТ №4

Название: Некорректная обработка пустой подстроки в contains
Статус: NEW
Приоритет: Minor
Компонент: StringUtils.contains()

Описание:
При вызове contains с пустой подстрокой метод должен возвращать False,
но текущая реализация сначала пытается выполнить index(""), 
затем попадает в except ValueError и возвращает False.

Это работает, но неявно и может ввести в заблуждение.

Как воспроизвести:
1. Вызвать contains("abc", "")
2. Ожидается: False
3. Фактически: False (но через обработку исключения)

ДЕФЕКТ №5

Название: trim удаляет только пробелы, но не табуляцию
Статус: NEW
Приоритет: Minor
Компонент: StringUtils.trim()

Описание:
Метод trim удаляет только символ пробела " ", но не удаляет табуляцию (\t).

Как воспроизвести:
1. Вызвать trim("\ttext")
2. Ожидается: "text" (логично ожидать удаление всех пробельных символов)
3. Фактически: "\ttext"
